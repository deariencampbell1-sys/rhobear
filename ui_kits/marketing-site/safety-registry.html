SECTION · SUBSTRATE · 13 / 20

In writing — what each agent can and can't do.

The safety registry is the written rulebook the crew runs against. Owner gates, agent registries, skill-drift sync, and the spread-out neo mode that catches drift before it merges.

OWNER GATES

You decide what the autoloop is allowed to do without you in the room.

TECH

Owner-configurable allow/deny matrices over tool calls, networks, package installs, secrets access, deploy targets, and irreversible actions.

AGENT REGISTRY

Every agent shows up with a name, a role, a scope, and a signature.

TECH

A signed registry of agent profiles binding role → tool scope → model runtime → operator-approved boundaries.

SKILL DRIFT SYNC

When the registered shape of a skill changes mid-build, RHOBEAR notices.

TECH

Versioned skill snapshots with diff detection and re-approval flows when capabilities change underneath an active goal.

SPREAD-OUT NEO MODE

Multi-agent runs cross-check each other before anything irreversible happens.

TECH

Distributed verification mode where independent reviewer instances re-evaluate critical actions before promotion.
